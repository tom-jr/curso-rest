    Ensina a criar um controller

é criado um diretorio na raiz da aplicação com o nome controller
 Então cria um arquivo com nome de PrimeiroController.java

    @RestController
    @RequestMapping(value = "/uri_controller" )
    public class PrimeiroController{

        @GetMapping(value = "/sub_uri")
        public String hoelloWorld(){

        return "Hello World!!!";
        }
    }

    Quando se tem um @ o que segue é denotado como uma annotation. neste exemplo, tres annotations são utilizadas

 -@RestController  --> informa ao compilador que essa classe s trata de um controllador rest. Ques estara recebendo as requisições que se direcionar para as
URIs que se direcionar que a representa no sistema.

    Toda e qualquer class RestController precisa ter sua URI declara para que assim possa receber requisições web. Essas URI é como um diretorio na class 
temos a URI pai, que vem informado na declaração da class com a annotation
 -@RequestMapping(value = "/URI")   O RequestMapping é o mapeamento de requisições. ele tem o paramentro value, que denota a URI Pai, então, os demais métodos
 que vinherem como metodos de requisições teram uma URI filho.
  Quando desejarmos fazer uma requisição para um metodo do do Controller utilizanos a URI filho precedido pela URI do pai. EX:
  
  http://localhost:8080/uri_pai/rui_filho
    
  

  as annotations dos metodos podem @GetMapping, @PutMapping, @PostMapping,@DeleteMapping, entre outros.

  cada um com sua função dentro de um CRUD [Create, Reade, Update, Delete]
